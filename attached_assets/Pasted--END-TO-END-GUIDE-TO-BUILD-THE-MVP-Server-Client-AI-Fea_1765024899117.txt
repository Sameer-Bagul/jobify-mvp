# 4Ô∏è‚É£ **Backend Implementation Guide (Server)**

---

## **4.1 Authentication**

Use JWT or NextAuth (if using Next.js API routes).
For backend API:

* `POST /auth/signup`
* `POST /auth/login`
* Encrypt passwords with bcrypt.
* Issue a JWT token valid for 7 days.

---

## **4.2 User Onboarding**

Endpoint:
`POST /user/onboarding`

Fields:

* gmail_id
* gmail_app_password (encrypt using AES)
* skills
* experience
* resume upload

Upload resume to:

* AWS S3
* Firebase Storage
* or local for MVP

---

## **4.3 Gmail SMTP Integration (Core Feature)**

Use **nodemailer**:

```js
const transporter = nodemailer.createTransport({
  service: "gmail",
  auth: {
    user: gmailId,
    pass: appPassword,
  }
});
```

### **Send Email Function**

```js
async function sendColdEmail({
  user,
  recruiter,
  renderedTemplate,
  resumePath
}) {
  return transporter.sendMail({
    from: user.gmail_id,
    to: recruiter.email,
    subject: `Application for ${recruiter.job_role || "a role"} at ${recruiter.company_name}`,
    html: renderedTemplate,
    attachments: [
      {
        filename: "Resume.pdf",
        path: resumePath
      }
    ]
  });
}
```

# 6Ô∏è‚É£ **Cold Email Automation Workflow**

1. User selects recruiter list (internal DB list).
2. Clicks **Send Emails**.
3. Backend:

   * Checks daily limit
   * Generates final email using template
   * Sends via Gmail SMTP
   * Logs each send
4. UI updates progress (e.g. ‚Äú12 / 20 sent today‚Äù).
